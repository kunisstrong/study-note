## 常见性能优化方案

1. 从发出请求到手刀相应的优化，比如DNS查询、HTTP长连接、HTTP2、HTTP压缩、HTTP缓存
2. 关键渲染路径优化、比如是否存在不必要的重绘和回流
3. 加载过程的优化，比如图片、视频等不同格式类型会有不同的使用后场景，在使用的过程中是否恰当
4. 构建优化，比如压缩合并、基于webPack构建优化方案等

## 指标

### RAIL性能模型

1. **响应(Response)：**应该尽可能凯苏的响应用户，应该在100ms以内响应用户输入
2. **动画(Animation)：**在展示动画的时候，每一帧应该以16ms进行渲染，这样可以保存动画效果一致性，逼并且避免卡顿
3. **空闲(Idle)：**当使用JavaScript主线程的时候，应该把任务划分成执行时间小于50ms的片段中区，这样可以释放线程以进行用户交互
4. **加载(Load)：**应该在小于1s的时候内加载完成你的网站，并可以进行用户交互

### 基于用户体验的性能指标

**FID：**首次输入延迟，从用户第一次与页面交互（点击链接、点击按钮）到浏览器能够响应该交互的时间

**FCP：**首次绘制来自DOM的内容的时间 

**TBT：**总阻塞时间

**TTI：**第一次达到可交互的时间

### Web Vital

**加载性能（LCP）：**显示最大内容元素所需时间

**交互性（FID）：**首次输入延迟时间

**视觉稳定性（CLS）**积累布局配置偏移
